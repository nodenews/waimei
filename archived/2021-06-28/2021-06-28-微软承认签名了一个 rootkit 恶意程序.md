[微软承认签名了一个 rootkit 恶意程序](https://www.solidot.org/story?sid=68143)
------

<div>
微软的代码签名流程<a href="https://www.bleepingcomputer.com/news/security/microsoft-admits-to-signing-rootkit-malware-in-supply-chain-fiasco/">能被攻击者的利用</a>。G Data 的安全研究人员本月早些时候收到了一个可能的误报：<a href="https://www.gdatasoftware.com/blog/microsoft-signed-a-malicious-netfilter-rootkit" target="_blank">由微软签名的 NetFilter 驱动程序</a>。从 Windows Vista 开始，在内核模式中运行的任何代码在公开发布前都需要经过微软的测试和签名，以确保操作系统的稳定性。没有微软签名的驱动程序默认是无法安装的。但安全研究人员在仔细分析后发现，Netfilter 是一个 rootkit 恶意程序，能自我更新和向攻击者的 IP 发送信息。WHOIS 查询显示，该 IP 地址属于<a href="https://www.tianyancha.com/company/2357743656">宁波卓智创新网络科技有限公司</a>。微软<a href="https://msrc-blog.microsoft.com/2021/06/25/investigating-and-mitigating-malicious-drivers/">表示</a>正在调查这起事件，它称攻击者通过 Windows Hardware Compatibility Program 项目递交驱动签名，它已经吊销了递交者的账号，评估其递交寻找恶意程序的其它信号。该恶意程序针对的是中国地区的游戏玩家。<p><img src="https://img.solidot.org//0/446/liiLIZF8Uh6yM.jpg" height="120" style="display:block"/></p>
</div>
